2 모델 서빙 (vLLM)

단일 GPU

CUDA_VISIBLE_DEVICES=0 \
VLLM_USE_DEEP_GEMM=0 \
VLLM_MOE_USE_DEEP_GEMM=0 \
uv run vllm serve "$HARNESS1_HF_MODEL" \
  --served-model-name harness-1 \
  --host 0.0.0.0 \
  --port 8000 \
  --tensor-parallel-size 1 \
  --max-model-len 32768 \
  --max-num-batched-tokens 16384 \
  --trust-remote-code \
  --moe-backend triton

다중 GPU

CUDA_VISIBLE_DEVICES=0,1 \
VLLM_USE_DEEP_GEMM=0 \
VLLM_MOE_USE_DEEP_GEMM=0 \
uv run vllm serve "$HARNESS1_HF_MODEL" \
  --served-model-name harness-1 \
  --host 0.0.0.0 --port 8000 \
  --tensor-parallel-size 2 \
  --max-model-len 32768 \
  --max-num-batched-tokens 16384 \
  --trust-remote-code \
  --moe-backend triton
ℹ️
--tensor-parallel-size는 가시 GPU 수와 동일하게. --moe-backend triton과 DeepGEMM 비활성화는 FlashInfer CUTLASS(nvcc 필요)·DeepGEMM 미지원 환경에서 흔한 기동 실패를 회피합니다.

헬스 체크

curl -sS http://127.0.0.1:8000/health   # 빈 성공 응답이면 준비 완료

3 스모크 테스트 (token IDs)

평가 하니스는 chat completions가 아니라 raw /v1/completions를 쓰고, vLLM이 생성 토큰 ID를 반환해야 합니다. 평가 전 반드시 확인하세요.

그림 3 · 평가 하니스는 토큰 ID 입출력을 사용 — 텍스트 API로는 동작하지 않음
python - <<'PY'
import json, urllib.request
payload = {
    "model": "harness-1",
    "prompt": "Say OK.",
    "max_tokens": 4,
    "temperature": 0.0,
    "stream": False,
    "return_token_ids": True,
}
req = urllib.request.Request(
    "http://127.0.0.1:8000/v1/completions",
    data=json.dumps(payload).encode("utf-8"),
    headers={"Content-Type": "application/json"}, method="POST",
)
with urllib.request.urlopen(req, timeout=120) as resp:
    data = json.loads(resp.read().decode("utf-8"))
choice = data["choices"][0]
tokens = choice.get("token_ids") or choice.get("tokens") or choice.get("text_token_ids")
print(choice.get("text", "").replace("\n", "\\n"))
print("token_ids_returned", bool(tokens))
PY
⚠️
token_ids_returned True가 출력될 때만 평가로 진행하세요. 아니면 더 최신 vLLM 빌드 필요.

5 평가 실행 · 모니터링 · 결과

실행

N_QUERIES=10
SEED=42
RUN_DIR=tmp/harness1_vllm_bcplus
mkdir -p "$RUN_DIR/trajectories"

export SAVE_TRAJECTORIES=1
export SAVE_FULL_TRAJECTORIES=1
export TRAJECTORY_SAVE_PATH="$RUN_DIR/trajectories"

PYTHONPATH=. uv run python inference/evaluate_harness1_vllm.py \
  --dataset browsecompplus \
  --split test \
  --collection-split test \
  --n-queries "$N_QUERIES" \
  --seed "$SEED" \
  --max-turns 40 \
  --temperature 1.0 \
  --max-tokens 2048 \
  --parallel 2 \
  --base-url http://127.0.0.1:8000/v1 \
  --model harness-1 \
  --partial-output "$RUN_DIR/partial_results.jsonl" \
  --output "$RUN_DIR/eval_results.json"

고정 질의로 평가하려면 --query-ids ...를 사용. 작은 N_QUERIES는 스모크용(분산 큼), 큰 값은 안정적 집계.

진행 모니터링

python - <<'PY'
from pathlib import Path
import json, statistics
run_dir = Path("tmp/harness1_vllm_bcplus")
rows = [json.loads(l) for l in (run_dir/"partial_results.jsonl").read_text().splitlines() if l.strip()]
print("completed", len(rows))
for key in ["recall","final_answer_recall","trajectory_recall","precision","turns"]:
    vals = [float(r.get(key,0.0) or 0.0) for r in rows]
    print(key, "mean", round(sum(vals)/max(len(vals),1),4),
          "median", round(statistics.median(vals),4) if vals else None)
print("errors", sum(1 for r in rows if r.get("error")))
PY

nvidia-smi   # GPU 사용률 확인

지표의미
recall최종 큐레이션 집합의 증거 문서 recall
final_answer_recall최종 답에 연결된 증거 recall
trajectory_recall탐색 trajectory 전체의 증거 recall
precision최종 큐레이션 집합의 precision
errors하니스 에러로 끝난 에피소드 수

증상해결
